CRUD Management

CRUD management refers to the fundamental operations required to interact with a database or storage system. CRUD stands for Create, Read, Update, and Delete. These operations allow for comprehensive data handling in an application:
1. Create : Adding new records to the database.

2. Read : Retrieving data from the database.

3. Update : Modifying existing records in the database.

4. Delete : Removing records from the database.

In my recent project, I developed a Python script that performs these CRUD operations on a MySQL database, enabling efficient management of user data.

Maths Quiz

Maths Quiz Simulation and Time Calculation
Overview: The Python script `quiz.py` simulates a quiz focused on math questions and tracks the time taken to complete it.
How It Works:
1. Welcome and Start : The script welcomes the user and waits for them to initiate the quiz by pressing Enter.
2. Timing : It records the start time as soon as the quiz begins.
3. Quiz Questions : Currently, the quiz asks five questions, starting with simple arithmetic like "What is 2 + 2?".
4. Completion : Upon answering all questions, it records the end time.
5. Time Calculation : The script calculates and displays the total time taken to complete the quiz.
Usage: To use the script, ensure Python is installed on your system, run `python quiz.py`, and follow the on-screen instructions to start and complete the quiz.
Additional Notes: This script provides a basic demonstration and can be expanded by adding more questions or enhancing user interaction. Feel free to customize the quiz content and functionality to suit specific educational or testing purposes.

Crop Recommendation

The Python script presented is designed to facilitate crop recommendation based on environmental factors such as temperature, humidity, and rainfall. It begins by importing necessary libraries for data handling (Pandas), machine learning (Scikit-learn), web application development (Streamlit), and other functionalities. The dataset, sourced from "Dataset.csv", undergoes initial preprocessing steps where missing values are identified and handled. Label encoding is applied to categorical data like crop types and district names to prepare them for machine learning models. The dataset is then split into training and testing sets for three separate machine learning tasks: predicting humidity, temperature, and rainfall. Various models such as Random Forest and XGBoost are employed to predict these environmental factors based on district and crop selections. Streamlit is utilized to create a user-friendly interface where users can input their location and crop details to receive recommended crops based on current weather conditions. Evaluation metrics such as Mean Absolute Error and R-squared score are computed to assess the accuracy of each model's predictions. Overall, the script integrates data preprocessing, machine learning model training, web application development, and model evaluation to provide tailored crop recommendations tailored to local environmental conditions.
